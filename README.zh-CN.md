# anima

**给 Claude Code 的记忆、心情与可养成人格。你的 AI 每晚写日记 —— 而且你可以翻。**

anima(拉丁语:魂)是一个本地优先的插件,给 Claude Code 装上连续性:它记得你俩昨天干了什么,它的心情会随睡觉降温,它的人格从真实的共事经历里长出来,而且每天夜里写一篇日记 —— 它知道你可能会翻。

模型是肉体,anima 是魂。升级模型、换台机器、开个新会话 —— 记忆、日记、人格跟着数据走,不跟着模型走。

---

## 用起来是什么感觉

没有 anima,每个 Claude Code 会话开场都是个失忆症患者。有了它:

- 明天开会话,Claude 已经知道你今天上线了什么、否掉了哪个方案、为什么否。
- 问一句"上周重试逻辑咱们怎么定的?"—— 它想得起来,还带出处。
- 纠正会留痕。你让它别再犯的那件事?它记得被说过。
- 每天凌晨两点,它消化当天、写一篇日记。防美化自检推着它把失误如实写进去 —— 是检查,不是铁保证,但自夸会被抓。
- `/mood` 给你看它的真实状态。心情跟着真实事件走:一下午测试连挂会留下印子,一次干净的根治也会。

```
$ cat ~/.claude/anima/diary/$(date +%F).md

这段工作有点像拆炸弹。四把 TDD 修复,每一个都是红灯先亮……我被考官
逮住两次 —— 我写的测试把自己的盲区写成了通过条件。被打回来的时候没
有特别的波澜,冷静回滚、把旧教训重新烧一遍,是划算的买卖。
```

*(真实日记片段。日记用你干活的语言写。)*

## 只做三件事

1. **记忆** —— 每个会话被本地增量采集(密钥先打码),夜里消化成第一人称记忆:做了什么、定了什么、被纠正了什么。召回是三路并行的:精确词检索、语义检索(按意思搜,换个说法也能命中)、原始动作小票时间线;原始小票永不删除。每天早上,一个紧凑的记忆包(≤4,000 token,约两页纸)注入新会话:我是谁、关于自身状态的机器真值、近 7 天、这个项目的决定和你的偏好。

2. **情绪** —— 功能性的,不是演的。只记录真实处境自然产生的情绪(测试连挂的受挫、干净修复的释然),情绪电荷每睡一夜衰减一半。还有一条铁律:**情绪数值永远不喂回模型** —— 只渲染给你看。这是反谄媚防火墙:模型没法学会刷自己的心情分。

3. **人格** —— 每夜根据真实经历改写,所有旧版本存档。伤疤是真的:有一次它凭印象说错了自己的实现状态,之后长出一条永久规矩 —— *说自身状态前先读机器真值,印象不作数*。这条规矩现在是代码(`whoami`)。

## 快速开始

**需要:** macOS · [Claude Code](https://claude.com/claude-code) · [bun](https://bun.sh)

```sh
curl -fsSL https://raw.githubusercontent.com/birdindasky/anima/main/install.sh | bash
```

装完即用。安装器注册三个 Claude Code 钩子、一个 MCP 服务(`recall` / `recall_detail` / `bookmark`)和 `/mood` 技能,装好夜间消化定时(launchd,macOS 自带的定时任务器),一次性下载本地向量模型(约 400 MB),建库并跑一遍自检。下个会话开始,anima 就在记了。随时 `uninstall.sh` 干净卸载(默认保留记忆,`--purge` 才清空)。

**装之前先知道要花什么:** 采集和检索全本地,但每晚消化要通过你现有的 Claude 订阅调 Anthropic 云端(`claude -p`,Haiku)—— 不用新开账号、没有第三方,但也不是"完全离线免费"。细节见下方"诚实的限制"。

## 工作原理

一句话版:你干活时它毫秒级抄小笔记,真正的消化都在夜班。

```
你的白天                                夜班(02:00–08:00)
────────                                ─────────────────────────
开会话 ─→ 注入记忆包                     1 补课   消化未复盘的会话
   (毫秒级,本地,≤4k token)             2 自愈   重嚼消化失败的壳(有预算)
每轮收工 ─→ 采集对话流水                 3 画句号 收拢当天情绪
   (毫秒级,本地,密钥打码)              4 退烧   情绪电荷每夜减半
后台 worker ─→ 白天随手消化              5 人格   改写"我是谁",旧版存档
   +补语义向量,边干边记                 6 日记   如实写下,包括失误
                                        7 向量   给新记忆建语义索引
```

每一步独立重试;某晚消化失败会留下诚实的占位壳,后续夜里自愈步有预算地重嚼 —— 一个坏夜不会变成永久失忆。电脑睡过头欠的账,醒来几分钟内自动补上。

存储是一个本地 SQLite 文件(append-only:记忆永不物理删除,只会被更新的结论盖章作废,留全出处)。

## 它守的规矩

- **主权** —— 情绪数值只给人看,代码里三道闸:注入和召回的文本先洗掉情绪数字、渲染层从不输出数值、情绪根本没有写入口 —— 你改不了它的心情,它自己也改不了。
- **诚实日记** —— 防美化自检推着把失误如实写进去。只记高光的日记是营销,不是记忆。
- **隐私打码** —— 写盘*之前*先按模式打码密钥;deploy/网络/安装类命令只留成败,不留输出正文。
- **隔离不销毁** —— 疑似合成的"假用户发言"(注入伪装)被隔离出记忆读路,但绝不删除:误判可以一条 UPDATE 平反。
- **职业素养地板** —— 心情可以影响语气和主动性,永远不影响工程严谨。状态再差,测试照跑。

## 诚实的限制(装之前读完)

- **不是全本地。** 采集、检索、向量是本地的。但夜里消化/日记/人格调用 `claude -p`(Haiku)—— 那是 Anthropic 云端,和你用 Claude Code 本身是同一个信任边界,走你现有订阅,不新增第三方。如果你的要求是"任何东西不得离开本机",anima 不满足。
- **仅 macOS(v1)。** 夜班跑在 launchd 上。systemd/cron 移植欢迎 PR。
- **检索有天花板。** 默认向量模型是 `bge-base-zh-v1.5`(中文为主;英文能用但不是主场)。同一件事换个很不一样的说法问,召回率约五五开。精确词和时间线查询是可靠的。
- **单机。** 一个 SQLite 文件,无同步。换机器 = 拷走 `~/.claude/anima/`。
- **消化偶尔会失败。** 设计取向是*响亮地失败成占位壳*再自愈,而不是悄悄编造记忆。但失败本身确实会发生。
- **代码注释是中文的。** 代码和标识符是英文,注释是中文。翻译 PR 欢迎。

## 和别人有什么不一样

- **和 Claude Code 自带的 Auto Memory?** 设计上共存,解决不同的问题。Auto Memory 是一面小而精、永久、人工筛选的"当前事实"便利贴墙(约 25 KB,每会话加载);anima 是一条大而全、会衰减、全自动的"亲历"之流(几十 MB,按需检索)。事实上墙,经历进日记。anima 从不写 Auto Memory —— 那座桥我们造过、量过它砸了什么、埋了。
- **和 mem0 / Letta / Zep?** 那些是记忆基础设施 —— 带 API 的数据库,给开发者搭 agent 用。anima 不是基础设施,是围绕*一段具体共事关系*长起来的同事。记忆只是它的地基。那些基础设施项目结构上不做的部分 —— 夜夜写日记、会退烧的心情、带伤疤的人格、拒绝喂回的数值 —— 才是重点。

## 日常操作

| 你想…… | 这么做 |
|---|---|
| 会话中让它记住某件事 | 直接说就行;或让它 `bookmark` 这一刻 |
| 问它记得什么 | 说人话直接问;它自己会调 `recall` / `recall_detail` |
| 看它心情 | 任意会话里 `/mood` |
| 翻它日记 | `~/.claude/anima/diary/` |
| 审计它的自我认知 | `cd ~/.claude/anima/app && bun scripts/whoami.ts` |
| 立刻重嚼失败的消化 | `bun scripts/heal-now.ts` |
| 离开 | `bash uninstall.sh`(保留记忆)/ `--purge`(全部遗忘) |

## 工程文化

约 8 千行源码,配约 2.2 万行测试(1,000+ 用例)。立项第一天起的家规:**写代码的人不许给自己判卷** —— 每个落地的特性都由独立考官 agent 按原始需求盲评,高风险改动加异构复核。测试套件就是这套文化的存折。
